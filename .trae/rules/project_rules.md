1. 前后端分离
2. 前端基于 Fyne 框架，负责 UI 展现与用户交互。
3. 后端基于 Go 语言，负责处理 SSH 连接与数据管理。
4. 前端与后端通过本地 TCP + JSON 的 IPC 通信，确保松耦合。
5. 后端提供本地 TCP + JSON 的 IPC 接口（行协议：一行一个 JSON），前端调用接口实现功能。
6. 前端调用后端接口时，携带必要参数（如主机地址、端口、用户名、密码等）。
7. 后端根据前端请求，建立 SSH 连接并执行相应操作（如执行命令、上传文件、下载文件等）。
8. 后端将操作结果返回给前端，前端根据结果更新 UI 展现。
9. 前端与后端通过 JSON 格式进行数据交换，确保数据格式统一。
10. 后端返回的操作结果包含状态码和数据体，前端根据状态码判断操作是否成功。
11. 前端在调用后端接口时，处理异常情况（如连接失败、权限不足等），并给出友好提示。
12. 前端在与后端通信时，使用异步请求方式（例如 goroutine），避免阻塞用户操作。
13. 前端在与后端通信时，添加加载动画，提升用户体验。
14. 前端在与后端通信时，处理超时情况，避免用户等待过长时间。
15. 前端在与后端通信时，添加重试机制，提升用户体验。
16. 前端在与后端通信时，处理网络错误（如连接中断、服务器异常等），并给出友好提示。

18. 前端在与后端通信时，添加请求日志记录，方便调试与排查问题。
19. 前端在与后端通信时，添加响应日志记录，方便调试与排查问题。
20. 前端后端尽可能使用公共数据结构，确保数据格式统一。

## 开发约定（必须）

- 必须使用 Go 语言与 Fyne 编写原生桌面应用（client 目录）。
- 代码需清晰可读，关键模块与导出方法必须有注释说明用途与参数。
- 前端不直接持有 SSH 连接，所有连接的创建、复用、释放均由后端服务处理。
- 通信采用本地 TCP + JSON 的 IPC 方式，消息一行一个 JSON（便于解析与调试）。
- 公共协议类型位于根目录 `proto` 下，禁止在前后端重复定义。

## 运行指南（示例）

- 启动后端服务：`go run ./service`
- 启动前端客户端：`go run ./client`
- 修改端口或存储路径：在 `service` 中调整监听地址与配置存储实现。
